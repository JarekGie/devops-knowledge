# PKM-Stack-Mastery

Notebook sluzy do praktycznego onboardingu do osobistego stacka poznawczego:
Obsidian + Claude + NotebookLM + vault + kontrakty + MOC + runbooki.

To nie jest source of truth. Notebook ma pomagac uczyc sie pracy na realnym vaultcie
i generowac artefakty robocze, ktore po review moga trafic do `findings/`.

## Jak zasilac notebook

- dodawaj skurated source packi z realnego vaulta, nie cale katalogi bez selekcji
- zaczynaj od `00-start-here/`, `90-reference/notebooklm/`, `40-runbooks/incidents/` i `20-projects/internal/llz/`
- dokladaj dokumentacje narzedzi tylko wtedy, gdy wspiera konkretna lekcje lub challenge
- jako pierwsze zrodlo zawsze dolacz `[[notebook-contract]]`

## Oczekiwane artefakty

- mikrolekcje osadzone w realnych notatkach vaulta
- checklisty i handoff notes
- contradiction check i gap analysis
- quizy, scenariusze problemowe i pakiety cwiczen
- robocze findings po review czlowieka

## Promocja do findings

- artefakt musi miec provenance
- wnioski musza byc oparte na zrodlach, bez zgadywania
- wynik po review czlowieka mozna strescic i zapisac do `findings/`
