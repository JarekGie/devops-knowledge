# Runbook — ALB 502 / 503

#networking #alb #runbook

## Symptom

ALB zwraca `502 Bad Gateway` lub `503 Service Unavailable`.

## Zakres

Jeden ALB / target group. Sprawdź czy dotyczy wszystkich lub wybranych targetów.

---

## Komendy diagnostyczne

```bash
ALB_ARN=arn:aws:elasticloadbalancing:...
TG_ARN=arn:aws:elasticloadbalancing:...

# Status targetów w target group
aws elbv2 describe-target-health --target-group-arn $TG_ARN

# Atrybuty target group (deregistration delay, healthcheck)
aws elbv2 describe-target-group-attributes --target-group-arn $TG_ARN

# Logi dostępowe ALB (jeśli włączone)
# S3: {bucket}/AWSLogs/{account}/elasticloadbalancing/{region}/{date}/
```

## Punkty decyzyjne

| Kod | Znaczenie | Przyczyna |
|-----|-----------|-----------|
| 502 | Zły gateway | Backend zwraca nieprawidłową odpowiedź HTTP lub padł |
| 503 | Service unavailable | Brak zdrowych targetów w target group |

## Diagnostyka 502

```bash
# Sprawdź czy backend działa i odpowiada na healthcheck
# Sprawdź port aplikacji vs port w target group
# Sprawdź czy SG backendu zezwala na ruch z ALB SG
```

## Diagnostyka 503

```bash
# Sprawdź target health
aws elbv2 describe-target-health --target-group-arn $TG_ARN \
  --query 'TargetHealthDescriptions[*].{target:Target.Id,state:TargetHealth.State,reason:TargetHealth.Reason}'
```

## Typowe przyczyny

| Przyczyna | Gdzie sprawdzić |
|-----------|----------------|
| App nie odpowiada na healthcheck path | healthcheck settings w TG |
| SG nie zezwala ALB → backend | security group rules |
| Port mismatch (TG vs kontener) | task definition vs TG port |
| App crash | logi, ECS task status |
| Zbyt szybki healthcheck | grace period, thresholds |

## Rollback

- Jeśli po deployu: rollback do poprzedniej task definition
- Jeśli infra: sprawdź ostatnie zmiany Terraform
