## 3. Root Cause

```
TagPolicyViolation — ECS nie może otagować ENI (Elastic Network Interface)
przy próbie uruchomienia taska Fargate.

Dowód (describe-tasks):
  StopCode:   TaskFailedToStart
  StopReason: Unexpected EC2 error while attempting to tag the network
              interface: TagPolicyViolation

Wszystkie trzy serwisy: desired=1, running=0, pending=0
ECS próbuje co ~13 min, za każdym razem TagPolicyViolation.

Hipoteza: nowa lub zmieniona Tag Policy w AWS Organizations wymaga
tagów których ECS nie dostarcza przy tworzeniu ENI.
```

## 6. Root Cause — pełna analiza

```
Tag Policy LLZ wymaga tagów Environment + Project na ec2:network-interface.
ECS Fargate tworzy ENI przy każdym starcie taska.
Serwisy mają propagateTags=SERVICE ale Tags=null → ENI bez tagów → violation.

Dlaczego api działa:
  rshop-prod-api-svc task startował 2026-04-17 20:16 (3 dni temu)
  ENI już istnieje — brak potrzeby tworzenia nowego → brak sprawdzenia policy

Dlaczego pozostałe padają:
  Deploymenty: backoffice 2026-04-19 16:02, frontend1 23:25, frontend2 12:21
  Dziś 09:xx — taski padły (nieznana przyczyna) i ECS próbuje je restartować
  Restart = nowy ENI = TagPolicyViolation

Policy dołączona do konta RShop (943111679945) bezpośrednio.
Wymagane wartości: Environment=[dev,qa,uat,prod,test,poc] Project=[rshop,...]
```

## 7. Fix (wymaga zmiany — READ-ONLY SESSION)

```bash
# KROK 1: Dodaj tagi do serwisów ECS (NOT read-only — wymaga zgody)
aws ecs tag-resource \
  --resource-arn arn:aws:ecs:eu-central-1:943111679945:service/rshop-prod-Klaster/rshop-prod-backoffice-svc \
  --tags key=Environment,value=prod key=Project,value=rshop \
  --region eu-central-1 --profile rshop

aws ecs tag-resource \
  --resource-arn arn:aws:ecs:eu-central-1:943111679945:service/rshop-prod-Klaster/rshop-prod-frontend-svc1 \
```
