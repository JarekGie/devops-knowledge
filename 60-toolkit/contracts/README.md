# Kontrakty — contracts/

Każdy plik = kontrakt jednej komendy toolkit.

**Należy tutaj:** definicje wejścia/wyjścia per komenda, JSON Schema kontraktów.  
**Nie należy tutaj:** implementacja (→ `commands/`), dokumentacja użytkownika (→ `60-toolkit/contracts-index.md`).

## Format pliku kontraktu

```markdown
# Kontrakt: {kategoria}.{komenda}

## Input

| Pole | Typ | Wymagane | Opis |
|------|-----|----------|------|

## Output

| Pole | Typ | Opis |
|------|-----|------|

## Przykład

\`\`\`json
{...}
\`\`\`
```
