# Projekty — Client

Projekty klienckie. Każdy klient = osobny podfolder.

**Należy tutaj:** kontekst klienta, dostępy, aktywne zadania, historia.  
**Nie należy tutaj:** runbooki generyczne (→ `40-runbooks/`), standardy (→ `30-standards/`).

## Struktura per klient

```
client/NAZWA-KLIENTA/
├── README.md         ← kim jest klient, zakres, kontakty
├── context.md        ← aktualny stan, blokery
├── decisions.md      ← decyzje podjęte w projekcie
├── next-steps.md     ← co dalej
└── links.md          ← repo, konsola, dokumenty
```
