# Area — Business

Model biznesowy DevOps-as-a-Service.

**Należy tutaj:** model pakietów, szablony ofert, struktura B2B, procesy klienckie.  
**Nie należy tutaj:** projekty klienckie (→ `20-projects/client/`), finanse (→ `70-finops/`).

## Model pakietowy

| Pakiet | Godziny / miesiąc | Zastosowanie |
|--------|------------------|-------------|
| Starter | 10h | audyt, onboarding |
| Standard | 20h | bieżące wsparcie |
| Extended | 40h | pełne wsparcie DevOps |
| Custom | negocjowane | projekty wdrożeniowe |

## Proces kliencki

```
1. Discovery call → zakres i potrzeby
2. Oferta → pakiet + stawka godzinowa
3. Umowa B2B → NDA + kontrakt usługowy
4. Onboarding → dostępy, repo, komunikacja
5. Realizacja → cykliczne raporty / call
6. Raport miesięczny → godziny + wyniki
```
